| ASM-014 | The AC-029 downstream-link gate (P8d, OQ-045) proxies "≥1 downstream artifact" as **≥1 ActionItem sourced from the decision** (`SourceType=Decision, SourceId`), and **exempts the supersession successor** from the gate. No schema change; the `IActionLinkDirectory` predicate + handler check are the only touch-points. | Approved | Tech Lead / Architect |
| ASM-015 | **P10c Traceability backbone assumptions.** (1) *No physical Artifact registry* — relationship edges self-describe via value snapshots `(Type, Id, Key, Title)` on both ends; matches docs/domain/search-and-traceability.md §2.1 SQL and reconciles ADR-0008's "registry" prescription via **ADR-0019** (amends 0008). (2) *AC-029 widened to a **curated** downstream set* — an edge satisfies the gate iff the decision is source of `recorded-as`/`resolves` or target of `implements`; upstream/lineage edges (`decided-by`/`derived-from`/`supersedes`) are excluded so a decision cannot be self-satisfied by its own topic. Superset of the Action-only gate → AC-029 stays Met. (3) *Title is a create-time snapshot* — may go stale on rename (accepted; deep-link fetches live data; keys never go stale). (4) *Owner-AiO edge-create deferred* — v1 create is Chairman/Secretary RBAC only (no ABAC handler for arbitrary artifact types; no UI). (5) *Panel shows explicit edges only* — projection of existing soft-refs (Decision→Topic, Action→source, Risk→subject) into edges is a later slice, and when built should be **read-time contract composition, not write-through** (one source of truth; keeps the upstream `decided-by` edge out of the AC-029 predicate). (6) *Edge endpoint types are NOT validated against the docs/domain/search-and-traceability.md §2.2 per-RelType matrix* — the create validator checks `IsInEnum` + source≠target only, not that (e.g.) `RecordedAs` runs Decision→ADR. Trusted-actor input (Chairman/Secretary only, mirrors ASM-014's `SourceId` posture). **Consequence:** a malformed edge (e.g. `Decision --RecordedAs--> Meeting`) can satisfy the AC-029 gate, since `DecisionHasDownstreamEdgeAsync` matches on source-type + RelType without checking the far-endpoint type. Acceptable at ≤20 on-prem users; the full 16-RelType type matrix is deferred. | Relationship edges self-describe via value snapshots (no physical Artifact registry, per docs/domain/search-and-traceability.md §2.1 and ADR-0019 amending ADR-0008); the AC-029 gate is widened to a curated downstream edge set (superset of the Action-only gate); titles are create-time snapshots; Owner-AiO create and soft-ref projection are deferred; endpoints are trusted-actor input not validated against the per-RelType matrix — all acceptable at ≤20 on-prem users. | If false: (1) a registry + per-module edge-writes would be needed (ADR-0001 cost); (2) the downstream RelType set is re-curated; (3) a read-time title-resolution seam is added; (4) an artifact-generic ABAC resource handler is built; (5) projection is added as its own slice; (6) the §2.2 per-RelType endpoint-type matrix is added to the validator. All are additive — no rework of the P10c edge schema or the Shared `ITraceabilityLinks` port. | Approved | Tech Lead / Architect |
| ASM-016 | **P10d Dependencies backbone assumptions.** (1) *Read-time composition, NO write-through mirror edge (OQ-046)* — a `Dependency` is stored ONCE (Dependencies module); it is NOT mirrored into the Traceability `Relationship` table. docs/domain/search-and-traceability.md §1.34/§1.1 previously implied write-through sync; per-module DbContexts share no transaction, so a mirror would risk dual-write drift for no benefit. The unified graph composes at read-time via the Dependencies module's artifact-scoped query. **Consequence:** the docs/domain/search-and-traceability.md §4.1 recursive-CTE impact traversal will NOT see dependency edges — **P10f must UNION the Dependencies table** (or compose in memory). Two-model boundary affirmed (docs/domain/domain-model.md §A.3; ADR-0008/0019). (2) *Kind = `{DependsOn,BlockedBy,Blocks,RelatesTo}`* — the design register's 4 chips; `Impacts` (docs/domain/domain-model.md) dropped as unused, `Blocks` added (FR-094 + design). (3) *`IsCrossStream` not modelled* — `(derived)` per docs/domain/domain-model.md, computed read-time in P10e from real stream *sets* (FR-095 needs a Topics+Membership stream-resolution contract, owned by P10e); P10d stores no flag. (4) *`Note` = plain `string?`* not `LocalizedString` (single free-text Notes box; user annotation ≠ system label; mirrors `Relationship.Notes`).
