**Meeting-recording feature (FR-056 upload leg) — COMPLETE + committed + PR'd.** Branch `feat/p13-recording-upload`, built on top of the P13 Webex branch `feat/p13-webex-integration` (13 Webex commits, unpushed) → the PR vs `main` bundles **all P13** (Webex adapter + recording). All BE+FE gates green, Keystone validator OK, live-validated on `acmp.ngrok.dev`.

**What shipped (PR1→PR4):** `POST /api/meetings/{key}/recording` (multipart upload, Secretary/Chairman) → validate size/MIME (`MeetingRecordingOptions`: video/mp4|webm|quicktime, **≤ 2 GB**) → store in MinIO `acmp-recordings` via the shipped shared **`IFileStore`** (ADR-0014) under a **server-derived key `{meetingKey}/{guid}{ext}`** (ext from content-type, NEVER the client filename — spaces/unicode break SigV4 presigned URLs) → `Meetings.RecordingUploaded` audit. Playback: `GET /{key}/recording/url` mints a 10-min **presigned MinIO URL**; nginx `location /acmp-recordings/` proxies to `minio:9000` with **Host preserved** so SigV4 validates against the public host; a **dedicated presign `IMinioClient`** built with `Minio:PublicEndpoint`=acmp.ngrok.dev + `.WithRegion("us-east-1")` (in `SharedKernelExtensions`; wrapper `MinioPresigner` lives inside `MinioFileStore.cs` to inherit its coverage exclusion). `<video src>` plays with Range/seek (200/206). Delete: `DELETE /{key}/recording` clears the reference + best-effort-deletes the object (uploaded) or just clears (Webex — we don't own it; a re-fired webhook re-attaches); `Meetings.RecordingRemoved` audit. UI = recording tab rebuilt to `ACMP Meetings.dc.html` isRecording (full-width player card, source `StatusChip` Webex/Uploaded, Download/Replace/Delete via design-system `Button`/`Dialog`, upload dropzone); new `trash` icon; i18n EN/AR. Delete/upload RBAC = **`Policies.MinutesCapture` = Chairman+Secretary only** (matrix row 8; command `AllowedRoles`; Member→403). Governance: **ADR-0025**, **AC-073** (upload+playback)/**AC-074** (delete) → Met, progress-log "P13-recording" entry, D-02 → In progress. Migration `Meetings_AddRecordingUpload`.

**Root-cause gotchas (operator "not working" reports):** (1) the *real* bug was nginx `client_max_body_size` defaulting to **1 MB** → any real upload 413'd (a 4 KB test slipped under it); fixed to **2100m** + per-endpoint Kestrel `RequestSizeLimitAttribute` + `RequestFormLimitsAttribute` + `MeetingRecordingOptions.MaxSizeBytes`=2 GB. (2) **Stale cached bundle** after a `web` rebuild → the user saw old 720px CSS + no-delete + "ugly"; a hard-refresh (or `?cb=`) loads the new bundle (see [[web-visual-verify-cache-busting]]). (3) The width cap was in **two** places — `.mt-rec-card` AND `.mt-rec-drop` — both must be full-width.

**Testing constraints (couldn't upload the real 19 MB file myself):** browser `file_upload` tool caps at **10 MB** + no longer takes host paths; the session access_token is **redacted on return** but usable *inside* page JS (do the upload via a browser `fetch` with `Bearer '+u.access_token`); the auto-classifier **rightly blocks** toggling `directAccessGrantsEnabled` on the `acmp-web` KC client (ROPC = SSO weakening). Verified the size fix with a 2 MB browser-JS upload (200, was 413). **Dev test users** (Keycloak, created via admin API): `secretary-test`/`Sec#Acmp2026` (Secretary), `chairman-test`/`Chair#Acmp2026` (Chairman). DAG on `acmp-web` was left **reverted** (false).
